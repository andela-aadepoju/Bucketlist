    This is an API for a bucket list service. A summary of the endpoints available on this API is represented in the table below

    End Points Functionalities

    End Point Function
    POST /auth/login: Logs a user in
    GET /auth/logout: Logs a user out
    POST /bucketlists: Creates a new bucket list
    GET /bucketlists: Lists all the created bucket lists
    GET /bucketlists/(id): Gets a single bucket list
    PUT /bucketlists/(id): Updates this single bucket list
    DELETE /bucketlists/(id): Deletes this single bucket list
    POST /bucketlists/(id)/items: Creates a new item in bucket list
    PUT /bucketlists/(id)/items/(item_id): Updates a bucket list item
    DELETE /bucketlists/(id)/items/(item_id): Deletes an item in a bucket list

    Details and usage of these endpoints can be found below.

    HTTP Status: 400

    NOTE: You must be logged in to create a bucket list. Also make sure you pass your authentication token with in your request header.

    To create a bucketlist, you need to send a POST request to “https://beebuckets.herokuapp.com/v1/bucketlists”

    Your request must have your authentication token passed along in the header. Other parameter include

    You need to pass your authentication token in your request header.

    Name: This is the bucketlist’s name. This is required

    Publicity: This will make your bucketlist public or not. It takes true or false as its value. To make your bucketlist public, set the value to true, and set it to false if you don’t want it to be public. By default, this is false.

    After successful creation, you will get a response similar to the responses.

    If any error occurs during the creation, You will get error messages of what went wrong. Example,responses

    Authentication

    Json Web Tokens(JWT), Token Based System was used for this API. With this, some end points are not accessible to unauthenticated users. Access control mapping is listed below.

    End Point and Public Access

    End Point Publicity
    POST /auth/login: TRUE
    GET /auth/logout: FALSE
    POST /bucketlists: FALSE
    GET /bucketlists: FALSE
    GET /bucketlists/(id): FALSE
    PUT /bucketlists/(id): FALSE
    DELETE /bucketlists/(id): FALSE
    POST /bucketlists/(id)/items: FALSE
    PUT /bucketlists/(id)/items/(item_id): FALSE
    DELETE /bucketlists/(id)/items/(item_id): FALSE

    Pagination

    This API is paginated such that users can specify the number of results they would like to have via a GET parameter limit.

    Example

    Request: GET https://beebuckets.herokuapp.com/v1/bucketlists?page=2&limit=20

    Response: 20 bucket list records belonging to the logged in user starting from the 21st bucket list .

    Searching by Name

    Users can search for bucket list by its name using a GET parameter q.

    Example

    Request: GET https://beebuckets.herokuapp.com/v1/bucketlists?q=bucket1

    Response: Bucket lists with the string “bucket1” in their name.
